## Security Policy

### Reporting a Vulnerability

If you discover a security vulnerability in this project, please do the following:

1. **Do not disclose it publicly.**
2. Open a private issue on GitHub or email vak9@student.le.ac.uk with details.
3. We will respond promptly and work to resolve the issue as quickly as possible.

### Best Practices

- Never commit secrets, credentials, or `.env` files to the repository.
- Always use environment variables for sensitive configuration.
- Keep dependencies up to date (`npm audit` regularly).
- Validate all user input on both client and server.
- Use HTTPS in production deployments.

Thank you for helping keep this project and its users safe!
